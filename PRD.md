## 🎯 项目背景

### 项目发起原因
- 中小学医务室日常工作需要数字化、智能化升级
- 减少医生手动登记工作量，提高效率
- 实现学生就诊全流程追踪，保障学生安全
- 积累医疗数据，支持流行病分析和决策

### 解决的痛点
1. **手动登记效率低** - 医生需要手动填写病历，耗时耗力
2. **学生流向不清晰** - 离开医务室后无法追踪去向
3. **药品库存管理困难** - 缺乏系统化管理，容易过期或短缺
4. **家长信息不对称** - 学生就诊后家长无法及时获知
5. **数据价值未挖掘** - 历史病例数据未有效利用

## 📝 核心需求

### 1. 身份验证
- 使用校园卡 + OAuth 统一接口验证（TODO，接入学校现有系统）
- 验证后获取学生姓名、学号、班级等基本信息

### 2. 电子病历系统
- 学生平板端填写病情（**引导式问答** + 人体图标选择 + **开放式输入**）
- 引导式问答为主，说不清楚时可切换开放输入
- 自动获取历史病例记录（如有）
- 生成结构化电子病历（无需打印）
- 病历包含：基本信息、主诉、诊断、处方、医嘱
- **病历储存无需监护人同意**（学校内部管理使用）
- **学生不可查询自己历史**（仅医生/家长可查看）
- **复诊提醒**：学生复诊时自动弹出首次就诊病历供医生参考

### 2.5 历史病历数字化（新增）
- 现有纸质病历扫描/拍照存档
- AI 辅助结构化录入（病历内容提取）
- 批量导入历史数据
- 新旧病历关联（同一学生）

### 3. AI 诊疗辅助
- 症状结构化（自由描述 → 标准化症状）
- 智能分诊（急诊/普通/建议转院）
- 药品推荐（基于症状 + 库存）
- 药物相互作用检查
- 流行病趋势检测

### 3.5 知识库系统

#### 药品知识库
- 数据来源：药品说明书、国家药监局数据库
- 内容：药品名称、规格、用法用量、禁忌症、不良反应、药物相互作用
- 更新方式：定期同步国家药监局数据

#### 疾病知识库
- 数据来源：临床诊疗指南、医学教科书
- 内容：常见疾病症状、诊断标准、治疗方案、鉴别诊断
- 覆盖范围：中小学常见疾病（感冒、发烧、外伤、过敏等）

#### 临床指南库
- 数据来源：卫健委发布的诊疗规范
- 内容：中小学医务室处理规范、转诊标准、应急预案

#### 历史病例库
- 数据来源：本校历史就诊记录
- 处理：脱敏处理（去除个人隐私）
- 用途：流行病分析、相似病例参考

### 3.6 AI 应用场景详解

#### 1️⃣ 患者端 - 症状采集
```
学生填写病情 → AI 结构化 → 标准化症状 (ICD-10)
```
- 引导式问答 + 开放输入 → AI 提取关键症状
- 避免学生描述不清
- 输出：标准化症状列表、严重程度评分

#### 2️⃣ 医生端 - 诊疗辅助
```
症状 + 历史病例 → AI 分析 → 诊疗建议 → 医生审核
```
- **智能分诊**：根据症状严重程度推荐优先级（急诊/普通/建议转院）
- **药品推荐**：基于症状 + 库存推荐药品（供医生参考）
- **药物相互作用检查**：自动检测处方中药物冲突
- **相似病例参考**：推荐历史相似病例及治疗方案
- ⚠️ **AI 不是替代医生，所有建议需医生审核确认**

#### 3️⃣ 管理端 - 数据分析
```
就诊数据 → AI 分析 → 流行病预警 + 报表
```
- **聚集性病例检测**：同班级 3 人以上相似症状自动预警
- **自动生成报表**：日报/周报/月报
- **流行病趋势预测**：基于历史数据分析趋势

#### 4️⃣ 知识库 - RAG 检索
```
医生问题 → 向量检索 → 知识库 + AI 总结 → 答案
```
- 药品信息查询（用法用量、禁忌症）
- 诊疗指南检索
- 历史病例参考

### 4. 药品库存管理
- 入库登记（扫码/手动）
- 出库记录（开药自动扣减）
- 库存预警
- 近效期预警
- 过期锁定

### 5. 安全追踪
- 学生流向登记（**由医生填写**：返回教室/寝室/离校/转诊）
- 留观学生管理
- 安全预警机制

### 6. 消息推送
- 嘉祥智慧教育平台公众号
- 推送对象：家长、老师
- 推送场景：就诊完成、留观通知、复诊提醒等

### 7. 报表系统
- 日报/周报/月报自动生成
- 钉钉 OA 上传

## 🏗️ 技术架构

### 系统架构
```
┌─────────────────────────────────────────────────────┐
│                    用户层                            │
│  学生终端 │ 医生终端 │ 管理后台 │ 公众号 │ 钉钉    │
└─────────────────────────────────────────────────────┘
                          ↓
┌─────────────────────────────────────────────────────┐
│                    应用层                            │
│  身份验证 │ 病历管理 │ 药品管理 │ AI 服务 │ 报表   │
└─────────────────────────────────────────────────────┘
                          ↓
┌─────────────────────────────────────────────────────┐
│                    数据层                            │
│  学生信息 │ 病历库 │ 药品库 │ 库存记录 │ 日志     │
└─────────────────────────────────────────────────────┘
```

### 技术选型
| 模块 | 技术选项 | 备注 |
|------|----------|------|
| 前端 | **React** + 响应式设计 | 医生端 + 管理后台，适配平板/电脑（组件多） |
| 患者端 | 平板 H5 网页 | 引导式问答 + 开放输入 |
| 后端 | **Go (Golang)** | API 服务（高性能、并发好、部署简单） |
| 数据库 | **PostgreSQL** | 结构化数据 |
| AI 服务 | **Qwen / Kimi / DeepSeek** | 症状结构化、诊疗建议 |
| 知识库 | **向量数据库 + 大模型 RAG** | 药品/疾病/指南检索 |
| 部署 | 云部署 或 本地服务器 | 可选 |
| 并发支持 | 2-5 名校医同时使用 | 低并发场景 |

### 技术选型理由
- **React**：组件生态丰富，开发效率高，社区活跃
- **Go**：编译型语言，性能优秀，并发模型简单，部署方便（单二进制文件）
- **PostgreSQL**：稳定可靠，支持复杂查询，适合医疗数据
- **向量数据库**：支持知识库语义检索（可选：Milvus/Chroma/pgvector）

### 推荐技术栈（具体实现）
| 模块 | 推荐方案 | 说明 |
|------|----------|------|
| 前端框架 | React 18 + TypeScript | 类型安全，减少 bug |
| UI 组件库 | Ant Design / MUI | 成熟的企业级组件库 |
| 状态管理 | Zustand / Redux Toolkit | 轻量级状态管理 |
| 后端框架 | Go + Gin / Echo | 轻量高性能 Web 框架 |
| ORM | GORM | Go 语言 ORM 库 |
| 向量数据库 | pgvector（PostgreSQL 插件） | 无需额外部署，直接用 PG |
| AI 接口 | 阿里云百炼 / DeepSeek API / Kimi API | 根据价格和效果选择 |
| 部署 | Docker + 云服务器/本地服务器 | 容器化部署，方便迁移 |

## 🔒 安全与隐私

### 数据保护
- 病历数据加密存储
- 敏感信息脱敏
- 数据访问日志记录
- 符合《个人信息保护法》《未成年人保护法》

### 权限管理
| 角色 | 可访问数据 |
|------|------------|
| 校医 | 全部就诊记录 |
| 班主任 | 本班学生记录 |
| 校领导 | 统计数据（不含隐私） |
| 家长 | 自己孩子的记录 |
| 后勤主任 | 药品库存数据 |

### 断网处理
- 支持断网时本地登记（数据暂存本地）
- 网络恢复后自动同步到服务器
- 避免断网时无法就诊登记

### 家长授权
- 首次使用需家长授权
- 授权协议明确数据使用范围

## ⚠️ 安全预警机制

| 预警类型 | 触发条件 | 通知方式 | 通知对象 |
|----------|----------|----------|----------|
| 留观超时 | 超过 2 小时 | 系统通知 | 医生 + 班主任 |
| 重复就诊 | 3 天内 2 次以上 | 系统通知 | 医生 |
| 药物相互作用 | 同时开多种药 | 弹窗提醒 | 医生 |
| 流行病趋势 | 同班级 3 人以上相似症状 | 系统通知 | 校医 + 校领导 |
| 未登记离开 | 进入后无离开记录 | 系统通知 | 校医 |
| 库存不足 | 低于安全库存 | 系统通知 | 后勤主任 |

| **症状分析** | 学生描述 → AI 提取关键症状 → 匹配可能疾病 | 辅助校医快速定位问题 |
| **病历生成** | 症状 + 检查结果 → AI 生成结构化病历草稿 | 减少医生手动填写 |
| **用药建议** | 症状 + 库存药品 → AI 推荐可用药品及剂量 | 结合库存，避免开缺货药 |
| **禁忌检查** | 处方药品 → AI 检查相互作用、过敏风险 | 用药安全保障 |
| **流行病预警** | 多病例分析 → AI 识别聚集性发病趋势 | 提前发现流感等传播 |
| **历史对比** | 当前症状 vs 该学生历史病例 → 提示复发风险 | 慢性病管理 |

**核心原则：AI 是辅助，最终决策权在校医**
