# 03｜前端架构（React + TypeScript）

## 1. 技术栈
- React 18
- TypeScript 5
- Vite
- React Router
- Axios
- Zustand（状态管理）
- Ant Design（企业后台 UI）

## 2. 目录建议
```text
frontend/src
├─app/               # 应用启动、路由、全局 Provider
├─modules/           # 业务模块（auth、visit、medicine、report）
├─shared/            # 复用能力（api、components、hooks、utils）
├─pages/             # 页面级组合
└─styles/            # 全局样式与主题
```

## 3. 分层约束
- `app` 只负责装配，不写业务逻辑；
- `modules` 承载业务状态与业务组件；
- `shared` 禁止依赖具体业务模块；
- 页面通过组合模块，不直接操作底层 API。

## 4. 路由规划（P0）
- `/student/checkin` 学生登记页
- `/doctor/visits` 医生接诊列表
- `/doctor/visit/:id` 就诊详情
- `/doctor/medicines` 药品管理
- `/admin/dashboard` 管理看板

## 5. API 调用规范
- 统一通过 `shared/api/http.ts` 发起请求；
- 统一处理 `token` 注入与错误拦截；
- 对业务接口按模块拆分（`modules/*/api.ts`）。

## 6. UI 与可用性原则
- 医生端优先信息密度和可读性；
- 学生端优先简洁引导、减少输入负担；
- 所有关键动作必须具备明确反馈（成功/失败/加载）。
