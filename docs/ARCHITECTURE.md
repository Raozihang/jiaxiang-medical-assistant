# 嘉祥智能医务室助手 - 技术架构文档

## 1. 系统架构

### 1.1 整体架构

```
┌─────────────────────────────────────────────────────────┐
│                      用户层                              │
│  患者端 (平板 H5)  │  医生端 (Web)  │  管理后台 (Web)   │
└─────────────────────────────────────────────────────────┘
                            ↓
┌─────────────────────────────────────────────────────────┐
│                      应用层                              │
│  身份认证 │ 病历管理 │ 药品管理 │ AI 服务 │ 报表系统   │
└─────────────────────────────────────────────────────────┘
                            ↓
┌─────────────────────────────────────────────────────────┐
│                      数据层                              │
│   PostgreSQL (主库 + pgvector) │ Redis (缓存/会话)     │
└─────────────────────────────────────────────────────────┘
```

### 1.2 部署架构

```
┌─────────────────────────────────────────────────────────┐
│                    Nginx (反向代理)                      │
│              静态资源 + API 路由 + SSL 终止               │
└─────────────────────────────────────────────────────────┘
            ↓                        ↓
┌──────────────────────┐  ┌──────────────────────┐
│   Frontend (React)   │  │   Backend (Go)       │
│   Vite 开发服务器     │  │   Gin 框架            │
│   端口：5173         │  │   端口：8080         │
└──────────────────────┘  └──────────────────────┘
                                    ↓
                        ┌──────────────────────┐
                        │   PostgreSQL         │
                        │   端口：5432         │
                        └──────────────────────┘
```

## 2. 技术栈说明

### 2.1 前端技术栈

| 技术 | 版本 | 用途 |
|------|------|------|
| React | 18.x | UI 框架 |
| TypeScript | 5.x | 类型安全 |
| Vite | 5.x | 构建工具 |
| Ant Design | 5.x | UI 组件库 |
| React Router | 6.x | 路由管理 |
| Axios | 1.x | HTTP 请求 |
| Zustand | 4.x | 状态管理 |

### 2.2 后端技术栈

| 技术 | 版本 | 用途 |
|------|------|------|
| Go | 1.21+ | 后端语言 |
| Gin | 1.9+ | Web 框架 |
| GORM | 1.25+ | ORM 库 |
| PostgreSQL | 15+ | 主数据库 |
| pgvector | 0.5+ | 向量检索（AI 知识库） |
| Redis | 7+ | 缓存/会话 |

### 2.3 AI 服务

| 服务 | 提供商 | 用途 |
|------|--------|------|
| 症状结构化 | Qwen/Kimi/DeepSeek | 自由文本 → 标准化症状 |
| 智能分诊 | Qwen/Kimi/DeepSeek | 症状 → 优先级建议 |
| 药品推荐 | Qwen/Kimi/DeepSeek | 症状 + 库存 → 推荐药品 |
| 药物相互作用检查 | 本地规则 + AI | 处方安全性检查 |
| 知识库检索 | RAG + pgvector | 药品/疾病/指南查询 |

## 3. 模块划分

### 3.1 前端模块

```
frontend/
├── src/
│   ├── app/              # 应用入口、路由配置
│   ├── modules/          # 业务模块
│   │   ├── patient/      # 患者端
│   │   ├── doctor/       # 医生端
│   │   └── admin/        # 管理后台
│   ├── shared/           # 共享组件、工具
│   │   ├── components/   # 通用组件
│   │   ├── hooks/        # 自定义 Hooks
│   │   ├── utils/        # 工具函数
│   │   └── api/          # API 请求封装
│   └── types/            # TypeScript 类型定义
└── public/               # 静态资源
```

### 3.2 后端模块

```
backend/
├── cmd/
│   └── server/           # 应用入口
├── internal/             # 内部包（不对外暴露）
│   ├── handler/          # HTTP 处理器
│   ├── service/          # 业务逻辑
│   ├── repository/       # 数据访问层
│   └── middleware/       # 中间件
├── pkg/                  # 公共包（可复用）
│   ├── models/           # 数据模型
│   ├── config/           # 配置管理
│   └── utils/            # 工具函数
└── configs/              # 配置文件
```

### 3.3 核心业务模块

| 模块 | 功能 | 接口 |
|------|------|------|
| 身份认证 | OAuth 登录、权限验证 | POST /api/auth/login |
| 病历管理 | 创建、查询、更新病历 | GET/POST /api/visits |
| 药品管理 | 库存管理、开药 | GET/POST /api/medicines |
| AI 服务 | 症状分析、分诊建议 | POST /api/ai/analyze |
| 报表系统 | 日报/周报/月报 | GET /api/reports |

## 4. 数据流设计

### 4.1 患者就诊流程

```
患者端填写病情
      ↓
AI 症状结构化
      ↓
医生接诊 → 查看历史病历
      ↓
医生诊断 → AI 辅助建议
      ↓
开具处方 → 药物相互作用检查
      ↓
药品出库 → 库存扣减
      ↓
填写学生流向
      ↓
推送家长通知
```

### 4.2 API 请求流程

```
客户端请求
      ↓
Nginx 反向代理
      ↓
Gin 路由匹配
      ↓
中间件（认证、日志、CORS）
      ↓
Handler 处理
      ↓
Service 业务逻辑
      ↓
Repository 数据访问
      ↓
PostgreSQL/Redis
      ↓
返回 JSON 响应
```

### 4.3 AI 服务调用流程

```
医生发起 AI 请求
      ↓
后端封装请求（症状 + 历史病例）
      ↓
调用 AI API（Qwen/Kimi/DeepSeek）
      ↓
解析 AI 响应
      ↓
本地规则校验（药物相互作用等）
      ↓
返回结构化建议
      ↓
医生审核确认
```

## 5. 关键技术决策

### 5.1 为什么选 Go？
- 高性能，并发模型简单
- 单二进制文件，部署方便
- 适合医疗系统（稳定、可靠）

### 5.2 为什么选 React？
- 组件生态丰富
- 开发效率高
- 企业级应用首选

### 5.3 为什么用 pgvector？
- 无需额外部署向量数据库
- 直接用 PostgreSQL
- 支持语义检索（AI 知识库）
