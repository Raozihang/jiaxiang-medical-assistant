# 10｜里程碑计划

> 规划基线日期：2026-02-23

## 1. 阶段目标
| 阶段 | 目标 | 核心交付 |
|---|---|---|
| P0 | 跑通核心业务闭环 | 前后端 MVP、核心接口、基础报表 |
| P1 | 提升效率与智能化 | AI 辅助、消息推送、自动化报表 |
| P2 | 增值能力建设 | 趋势分析、健康报告、扩展硬件 |

## 2. 建议排期（可调整）
| 周期 | 重点 |
|---|---|
| 第 1-2 周 | 工程初始化、文档、基础模型与 API |
| 第 3-4 周 | 就诊流程与药品库存主链路 |
| 第 5-6 周 | 报表与管理功能完善、联调测试 |
| 第 7 周+ | P1 能力迭代（AI 与消息） |

## 3. P0 里程碑
- M1：完成前后端脚手架与规范；
- M2：完成就诊与库存核心接口；
- M3：完成前端基础页面与联调；
- M4：完成冒烟测试与阶段验收。

## 4. 风险与应对
| 风险 | 影响 | 应对 |
|---|---|---|
| 学校 OAuth 对接延迟 | 登录流程受阻 | 先用本地 mock 鉴权，后续切换 |
| 历史病历格式不统一 | 数据迁移延期 | 先定义标准导入模板 |
| 外部消息平台联调周期长 | 推送功能延期 | P0 不阻塞核心链路 |
| AI 成本与效果不稳定 | P1 交付风险 | 抽象供应商接口，支持替换 |
