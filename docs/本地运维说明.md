# 智慧医务室系统 运维说明

## 1. 当前线上信息

- 站点地址：`https://ai.cdjxfls.com`
- HTTP 会自动跳转到 HTTPS
- 后端健康检查：`https://ai.cdjxfls.com/api/v1/healthz`
- 当前数据模式：`db`

## 2. 当前登录账号

- 管理员：`admin / 123456`
- 医生：`doctor / 123456`

说明：
- 这是应用登录密码，不是服务器 root 密码。
- 数据库密码和 JWT 密钥仍是强密码，没有改成弱口令。

## 3. 服务器关键信息

- SSH：`root@103.116.246.203 -p 44274`
- 系统：`Ubuntu 24.04`
- Nginx：宝塔环境，配置目录为 `/www/server/panel/vhost/nginx/`
- 前端站点目录：`/www/wwwroot/ai.cdjxfls.com`
- 项目源码目录：`/opt/jiaxiang-medical-assistant/src`
- 后端服务名：`medical-assistant`

## 4. 数据库信息

- 应用实际连接的 PostgreSQL 端口：`5433`
- 数据库名：`medical_assistant`
- 数据库用户：`medical_app`

说明：
- 服务器上原本还有一套宝塔 PostgreSQL 在 `5432`
- 当前应用不是连 `5432`，而是连 `5433`

## 5. 常用运维命令

查看后端状态：

```bash
systemctl status medical-assistant --no-pager -l
```

重启后端：

```bash
systemctl restart medical-assistant
```

查看后端日志：

```bash
tail -n 100 /var/log/medical-assistant/backend.log
tail -n 100 /var/log/medical-assistant/backend-error.log
```

检查 Nginx 配置并重载：

```bash
nginx -t
systemctl reload nginx
```

检查健康接口：

```bash
curl -k https://127.0.0.1/api/v1/healthz
curl -s https://ai.cdjxfls.com/api/v1/healthz
```

查看数据库表：

```bash
sudo -u postgres psql -p 5433 -d medical_assistant -c "\dt"
```

## 6. HTTPS 证书

- 证书路径：`/etc/letsencrypt/live/ai.cdjxfls.com/fullchain.pem`
- 私钥路径：`/etc/letsencrypt/live/ai.cdjxfls.com/privkey.pem`
- 续期后会自动执行：`systemctl reload nginx`

## 7. 更新前端标题

本地代码里的默认网页标题已经改为：

```text

```

相关文件：

- `frontend/index.html`

如果之后要重新发版，常用流程：

```bash
cd /opt/jiaxiang-medical-assistant/src/frontend
VITE_API_BASE_URL=/api/v1 VITE_APP_TITLE='智慧医务室系统' npm run build
rm -rf /www/wwwroot/ai.cdjxfls.com/*
cp -r dist/* /www/wwwroot/ai.cdjxfls.com/
chown -R www:www /www/wwwroot/ai.cdjxfls.com
systemctl reload nginx
```

## 8. 故障排查优先顺序

1. 先看 `https://ai.cdjxfls.com/api/v1/healthz` 是否返回 200。
2. 如果前端白屏，先确认 `/www/wwwroot/ai.cdjxfls.com` 下文件是否完整。
3. 如果接口异常，先看 `medical-assistant` 服务状态和日志。
4. 如果数据库异常，确认应用仍连接 `5433` 而不是 `5432`。
