# 08｜测试策略

## 1. 测试目标
- 保证核心业务链路稳定；
- 快速回归高风险功能（就诊、库存、权限）；
- 在低并发场景下保证可用性与正确性。

## 2. 前端测试
- 单元测试：组件与工具函数；
- 集成测试：页面与状态流转；
- E2E（后续）：关键流程回归（登记→接诊→完成）。

建议工具：Vitest + React Testing Library + Playwright。

## 3. 后端测试
- 单元测试：service/repository 逻辑；
- 集成测试：handler + DB（测试库）；
- 契约测试（后续）：前后端接口字段契约。

建议命令：
```bash
go test ./... -v
```

## 4. 测试分层比例（参考）
- 单元测试：70%
- 集成测试：20%
- 端到端测试：10%

## 5. 验收回归清单（P0）
- 学生登记成功；
- 医生录入病历并更新去向；
- 开药后库存正确扣减；
- 错误请求返回标准错误结构；
- 报表接口可返回基础统计。
