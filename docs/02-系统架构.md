# 02｜系统架构

## 1. 架构概览
系统采用前后端分离架构，面向低并发（2-5 名校医并发）与高可维护性设计。

```text
客户端层
├─学生端（H5）
├─医生端（Web）
└─管理端（Web）

服务层
├─API Gateway（Gin）
├─业务模块（病历、药品、报表、通知）
└─AI/RAG 适配层（后续接入）

数据层
├─PostgreSQL（主数据）
├─Redis（缓存/会话）
└─对象存储（扫描件、附件）
```

## 2. 业务域划分
- **身份域**：校园卡/OAuth 登录、会话、权限；
- **就诊域**：登记、问诊、诊断、处方、医嘱、复诊提醒；
- **药品域**：入库、出库、库存与效期预警；
- **安全追踪域**：去向登记、留观管理、异常告警；
- **报表域**：就诊统计、药品报表、趋势分析。

## 3. 核心链路
1. 学生身份确认并提交病情；
2. 医生端接诊并生成电子病历；
3. 开药后自动扣减库存；
4. 医生登记学生离开去向；
5. 系统沉淀数据并驱动报表/预警。

## 4. 非功能要求
- **安全性**：敏感字段最小暴露、关键操作可审计；
- **可用性**：支持断网暂存后补同步（P1）；
- **可维护性**：模块化分层，接口与数据模型可演进；
- **可扩展性**：预留 AI 服务与知识库接入边界。

## 5. 外部集成边界
| 外部系统 | 用途 | 当前策略 |
|---|---|---|
| 学校 OAuth | 身份认证 | 先抽象 `AuthProvider`，后续落地接入 |
| 嘉祥智慧教育平台 | 通知推送 | 预留消息网关接口 |
| 钉钉 OA | 报表上报 | 预留导出与上传能力 |
| 模型 API | AI 辅助 | 通过 `AIClient` 统一适配 |
