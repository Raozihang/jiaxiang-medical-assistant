# 开发规范

## 1. 代码风格

### 1.1 TypeScript 代码风格

```typescript

// .prettierrc
{
  "semi": true,
  "singleQuote": true,
  "tabWidth": 2,
  "trailingComma": "es5"

// .eslintrc.js
module.exports = {
  extends: [
    'eslint:recommended',
    'plugin:@typescript-eslint/recommended',
    'plugin:react/recommended'
  ],
  parser: '@typescript-eslint/parser',
  rules: {
    'no-console': 'warn',
    'prefer-const': 'error'
  }

```

### 1.2 Go 代码风格

```go

// 格式化代码
gofmt -w .

// 检查代码
go vet ./...

// 静态分析
golint ./...
```

## 2. 命名规范

### 2.1 TypeScript 命名

```typescript
// 类和接口：PascalCase
interface UserResponse { }
class UserService { }

// 变量和函数：camelCase
const userName = 'John';
function getUserInfo() { }

// 常量：UPPER_SNAKE_CASE
const MAX_RETRY_COUNT = 3;

// 文件命名：kebab-case
// user-service.ts
// visit-list.tsx
```

### 2.2 Go 命名

```go
// 导出标识符：PascalCase
type VisitRequest struct { }
func CreateVisit() { }

// 非导出标识符：camelCase
type visitData struct { }
func createVisit() { }

// 常量：PascalCase 或 camelCase
const MaxRetryCount = 3
```

## 4. Git 提交规范

### 4.1 Commit Message 格式

```
<type>(<scope>): <subject>

<body>

<footer>
```

### 4.2 Type 类型

| Type | 说明 |
|------|------|
| feat | 新功能 |
| fix | 修复 bug |
| docs | 文档更新 |
| style | 代码格式（不影响功能） |
| refactor | 重构 |
| test | 测试相关 |
| chore | 构建/工具/配置 |

### 4.3 提交示例

```bash
# 新功能
git commit -m "feat(visits): 创建就诊记录功能"

# 修复 bug
git commit -m "fix(auth): 修复登录 token 过期问题"

# 文档更新
git commit -m "docs: 更新 API 文档"

# 重构
git commit -m "refactor(api): 重构错误处理逻辑"
```

## 5. 分支管理

### 5.1 分支命名

```
main              # 主分支（生产）
develop           # 开发分支
feature/xxx       # 功能分支
bugfix/xxx        # 修复分支
hotfix/xxx        # 紧急修复
release/v1.0.0    # 发布分支
```

### 5.2 分支流程

```bash
# 创建功能分支
git checkout develop
git checkout -b feature/visit-module

# 开发完成后提交
git add .
git commit -m "feat: 完成就诊模块"

# 推送到远程
git push origin feature/visit-module

# 创建 Pull Request 到 develop
```

## 6. Code Review 清单

### 6.1 代码质量

- [ ] 代码是否有注释？
- [ ] 命名是否清晰？
- [ ] 函数是否过长（<50 行）？
- [ ] 是否有重复代码？
- [ ] 是否有潜在 bug？

### 6.2 测试覆盖

- [ ] 是否添加了单元测试？
- [ ] 测试用例是否覆盖边界情况？
- [ ] 测试是否通过？

### 6.3 文档更新

- [ ] 是否更新了 README？
- [ ] 是否更新了 API 文档？
- [ ] 是否更新了 CHANGELOG？

### 6.4 安全性

- [ ] 是否有 SQL 注入风险？
- [ ] 是否有 XSS 风险？
- [ ] 敏感信息是否加密？
- [ ] 权限检查是否完整？

## 9. 错误处理

### 9.1 前端错误处理

```typescript

  await api.createVisit(data);
} catch (error) {

    // 跳转登录
  } else {
    // 显示错误提示

  }
}
```
