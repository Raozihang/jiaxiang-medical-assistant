# 09｜安全与隐私

## 1. 原则
- 最小权限原则；
- 默认拒绝原则；
- 敏感数据最小化采集与展示；
- 全链路可审计。

## 2. 访问控制
- 基于角色的访问控制（RBAC）：学生、校医、班主任、校领导、后勤；
- 接口层统一鉴权与权限校验；
- 后台高危操作（删除、批量导入）保留审计日志。

## 3. 数据保护
- 传输加密：HTTPS；
- 存储加密：数据库磁盘加密 + 备份加密；
- 脱敏展示：按角色隐藏不必要隐私字段。

## 4. 审计与追踪
- 记录登录、查询、修改、导出等关键行为；
- 每个请求写入 `request_id`；
- 审计日志保留策略建议不少于 180 天。

## 5. 合规与边界
- 参考《个人信息保护法》《未成年人保护法》；
- AI 建议仅做辅助，不替代医生决策；
- 外部模型调用需控制脱敏与最小必要信息。

## 6. 安全事件响应（建议）
1. 发现异常并分级；
2. 快速隔离影响范围；
3. 追溯日志与根因定位；
4. 修复与补丁发布；
5. 复盘与规范更新。
