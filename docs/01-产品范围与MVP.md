# 01｜产品范围与 MVP

## 1. 目标与背景
根据 `PRD.md`，本项目聚焦校园医务室数字化，目标是：
- 提升接诊与病历登记效率；
- 实现就诊过程可追踪；
- 建立药品库存与预警能力；
- 为校医与管理层提供数据决策支撑。

## 2. 角色与核心场景
| 角色 | 核心场景 | 关键能力 |
|---|---|---|
| 学生 | 平板填报病情 | 引导式问诊、快速登记 |
| 校医 | 接诊与处理 | 病历录入、诊断、处方、流向登记 |
| 家长 | 了解就诊情况 | 接收就诊通知、用药提醒 |
| 班主任/校领导 | 管理与决策 | 聚合数据查看、预警接收 |
| 后勤主任 | 药品管理 | 库存监控、补货决策 |

## 3. MVP（P0）范围
### 3.1 患者端
- 校园卡/OAuth 身份验证（先预留接口）；
- 引导式病情填写；
- 离开去向登记。

### 3.2 医生端
- 患者信息与历史就诊记录查看；
- 电子病历生成（主诉、诊断、处方、医嘱）；
- 复诊提醒；
- 药品出入库与库存预警（简化版）。

### 3.3 管理端
- 基础报表（日/周/月）生成；
- 知识库管理后台（导入入口与管理能力）。

### 3.4 数据迁移
- 纸质病历扫描归档；
- 历史病历结构化导入（可 AI 辅助）；
- 新旧病历关联。

## 4. P1/P2 演进范围
- P1：AI 症状结构化、智能分诊、药物相互作用、消息推送、完整报表自动化。
- P2：流行病趋势分析、班级健康报告、扩展硬件能力（如眼镜找药）。

## 5. 非目标（当前阶段）
- 多校区支持；
- 值班排班与交接班系统；
- 学生自助查询历史病历；
- 完整离线运行。

## 6. P0 验收标准（DoD）
- 前后端工程可本地启动；
- 核心业务链路可跑通（登记→接诊→处置→离开）；
- 药品库存具备最小可用管理能力；
- 关键操作具备日志与错误提示；
- 文档可指导新成员 1 天内完成开发环境搭建。
